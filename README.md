# HOL Manager - огляд проєкту

Це програма для управління персоналом, створена за допомогою сучасного набору веб-технологій.

## Основний стек технологій

- **Фреймворк:** **Next.js (App Router)** - забезпечує швидку та оптимізовану роботу програми.
- **Мова:** **TypeScript** - для надійного та масштабованого коду.
- **UI Компоненти:** **shadcn/ui** - колекція готових, стильних та доступних компонентів.
- **Стилізація:** **Tailwind CSS** - для швидкої та консистентної розробки інтерфейсу.
- **База даних:** **Firebase Realtime Database** - для зберігання даних у реальному часі.
- **Іконки:** **Lucide React** - для чистого та сучасного набору іконок.
- **Діаграми:** **Recharts** - для візуалізації даних.
- **Форми:** **React Hook Form** - для керування формами та валідації.
- **Штучний інтелект:** **Genkit (від Google)** - для інтеграції генеративних моделей ШІ.

---
## Функціональні Можливості

### Загальний Опис

**HOL Manager** — це комплексна CRM-система, розроблена для автоматизації та оптимізації процесів управління персоналом. Вона дозволяє вести повний цикл життя співробітника в компанії: від найму до звільнення, включаючи управління відпустками, відстеження присутності, видачу робочого одягу та генерацію необхідної документації. Система розроблена з акцентом на швидкість, інтуїтивність та гнучкість налаштувань.

---

### Детальний Опис Функцій по Вкладках

#### 1. **Працівники Активні (`aktywni`)**
Центральна панель для управління всіма активними співробітниками.
- **Перегляд та Редагування:** Відображення працівників у вигляді таблиці (десктоп) або карток (мобайл) з можливістю редагування будь-якого поля.
- **Додавання Нових Працівників:** Інтуїтивна форма для швидкого додавання нових співробітників з валідацією даних.
- **Потужна Фільтрація:** Можливість миттєво фільтрувати список за іменем, прізвищем, номером картки, відділом, керівником, посадою та національністю.
- **Фільтр за Датою:** Фільтрація працівників за діапазоном дати працевлаштування.
- **Імпорт та Експорт:** Масовий імпорт нових працівників з файлу Excel та експорт відфільтрованого списку у формат `.xlsx`.
- **Масові Операції:** Можливість видалення всіх дат працевлаштування або всіх працівників з системи (з підтвердженням).
- **Генерація Підсумку (AI):** Використання ШІ для автоматичного створення короткого текстового резюме по обраному працівнику польською мовою.

#### 2. **Працівники Звільнені (`zwolnieni`)**
Архів звільнених працівників для ведення історії та можливого відновлення.
- **Перегляд Історії:** Доступ до повного списку звільнених співробітників з датами працевлаштування та звільнення.
- **Фільтрація:** Аналогічні до активних працівників можливості фільтрації.
- **Відновлення Працівника:** Можливість в один клік відновити звільненого працівника, повернувши його до списку активних.
- **Імпорт та Експорт:** Спеціалізований функціонал для імпорту та експорту саме звільнених працівників.

#### 3. **Планування (`planowanie`)**
Стратегічна панель для візуалізації майбутніх кадрових змін.
- **Плановані Звільнення:** Окремий список працівників, для яких встановлено майбутню дату звільнення.
- **Працівники у Відпустці:** Список співробітників, які перебувають у відпустці *прямо зараз*.
- **Майбутні Відпустки:** Список запланованих відпусток, які ще не почалися.
- **Візуальне Розділення:** Чітка структура з трьох колонок для швидкого аналізу ситуації.

#### 4. **Відвідуваність (`odwiedzalnosc`)**
Інтерактивний календар для відстеження присутності та неявок.
- **Візуальний Календар:** Для кожного працівника відображається календар на обраний місяць, де можна в один клік позначити день як неявку.
- **Статистика в Реальному Часі:** Автоматичний підрахунок загальної кількості неявок, відсотка відсутності по кожному працівнику та загалом по відділах.
- **Фільтрація та Навігація:** Зручне перемикання між місяцями та роками, а також фільтрація за відділом та іменем.
- **Звіт по Відділах:** Наочна діаграма, що показує відсоток неявок у розрізі відділів.
- **Список Неявок:** Можливість переглянути та скопіювати зведений список усіх неявок за місяць.

#### 5. **Статистика (`statystyki`)**
Аналітична панель з інтерактивними діаграмами для глибокого аналізу структури персоналу.
- **Розподіл за Відділами:** Багаторівнева діаграма, що дозволяє "провалитися" від загального списку відділів до конкретних керівників, а потім — до посад у їх підпорядкуванні.
- **Розподіл за Національністю та Посадами:** Дві окремі діаграми, що наочно показують структуру персоналу за цими критеріями.
- **Інтерактивність:** Клік на будь-який сегмент діаграми відкриває спливаюче вікно зі списком відповідних працівників, який можна скопіювати.

#### 6. **Видача Одягу (`wydawanie-odziezy`)**
Модуль для реєстрації та обліку виданого робочого одягу.
- **Створення Акту Видачі:** Форма для вибору працівника та додавання елементів одягу зі списку (з можливістю вказати кількість).
- **Друк Підтвердження:** Автоматична генерація та друк формалізованого документа (А4) "Підтвердження видачі", готового для підпису.
- **Історія Видач:** Для кожного працівника ведеться повна історія всіх попередніх видач з можливістю повторного друку будь-якого акту.

#### 7. **Карти обігові (`karty-obiegowe`)**
Інструмент для створення обхідних листів при звільненні.
- **Генерація Карти:** Вибір працівника (активного або звільненого) та миттєва генерація карти.
- **Друк:** Вивід на друк стандартизованої "Карти обігової" у форматі А4.
- **Історія:** Ведення архіву всіх згенерованих карт для кожного працівника з можливістю повторного друку.

#### 8. **Відбитки Пальців (`odciski-palcow`)**
Модуль для планування візитів до відділу кадрів для здачі біометричних даних.
- **Планування Візиту:** Проста форма для вибору працівника та призначення дати й часу візиту.
- **Автоматичні Нагадування (AI):** Система автоматично перевіряє заплановані візити та за 2 дні створює сповіщення у внутрішній системі нотифікацій.

#### 9. **Брак Логовання (`brak-logowania`)**
Інструмент для фіксації інцидентів, коли працівник не відмітився на вході/виході.
- **Створення Запису:** Форма для вибору працівника, дати інциденту, вказання годин роботи та причини (забув/відсутня картка).
- **Друк Пояснювальної:** Генерація та друк офіційного бланку "Oświadczenie", який працівник та керівник можуть підписати.
- **Історія Інцидентів:** Ведення списку всіх зареєстрованих випадків.

#### 10. **Конфігурація (`konfiguracja`)**
Центр управління всіма довідниками, які використовуються в системі. Це дозволяє гнучко налаштовувати випадаючі списки та опції в усьому додатку без необхідності змінювати код.
- **Управління Списками:** Можливість додавати, редагувати та видаляти елементи у наступних довідниках:
  - **Відділи (`Działy`):** Назви всіх відділів компанії.
  - **Посади (`Stanowiska`):** Перелік усіх доступних посад.
  - **Керівники (`Kierownicy`):** Список осіб, які можуть бути призначені керівниками.
  - **Національності (`Narodowości`):** Список національностей для зручного вибору.
  - **Елементи одягу (`Odzież`):** Перелік робочого одягу, що видається працівникам.
- **Масове Додавання:** Дуже зручна функція, що дозволяє додавати відразу кілька нових елементів, просто вставляючи їх списком, де кожен елемент знаходиться на новому рядку.
- **"Розумне" Редагування:** При зміні назви будь-якого елемента (наприклад, перейменуванні відділу з "Продукція" на "Виробництво"), система автоматично оновить це значення у всіх працівників, до яких воно було прив'язане. Це забезпечує цілісність та актуальність даних у всій системі.

## Підхід до Адаптивного Дизайну (Responsive Design)

Для того, щоб програма виглядала добре та була зручною у використанні на будь-якому пристрої, від великого монітора до маленького смартфона, використовується кілька ключових технологій та підходів:

### 1. Tailwind CSS (Основний інструмент)

- **Адаптивні префікси:** Це найголовніша техніка. Я використовую спеціальні префікси, такі як `sm:`, `md:` та `lg:`, щоб застосовувати різні стилі для різних розмірів екрана. Наприклад, клас `grid-cols-1 lg:grid-cols-3` означає: "за замовчуванням показувати елементи в одну колонку, але на великих екранах (lg) — у три". Це дозволяє створювати гнучкі сітки, які автоматично перебудовуються.
- **Flexbox та Grid Layout:** Активно використовуються ці сучасні CSS-технології. Вони дозволяють елементам гнучко розтягуватися, стискатися та переноситися на новий рядок, залежно від доступного простору. Це ідеально підходить для створення "гумових" інтерфейсів.
- **Приховування елементів:** Іноді деякі елементи не потрібні на маленьких екранах. За допомогою класів Tailwind, таких як `hidden lg:block`, я можу повністю приховати елемент на мобільних пристроях (`hidden`) і показати його тільки на великих екранах (`lg:block`).

### 2. Умовний рендеринг компонентів (React)

- **Спеціальний хук `useIsMobile`:** Створено невелику утиліту `src/hooks/use-mobile.tsx`, яка визначає, чи є екран мобільним (меншим за 1024 пікселі).
- **Різні компоненти для різних екранів:** Залежно від результату `useIsMobile`, програма може показувати абсолютно різні компоненти. Найкращий приклад — це навігація. На великому екрані ви бачите бічну панель (`AppSidebar`), а на мобільному — нижню панель з іконками (`AppBottomNav`). Це дозволяє створювати кардинально різні, але оптимальні інтерфейси для кожного типу пристрою. Такий самий підхід застосовано для сторінок працівників, де на мобільних пристроях відображаються картки замість таблиці.

### 3. Адаптивні одиниці виміру

- Використовуються відносні одиниці, такі як відсотки (%) або `rem`, які масштабуються відносно базового розміру шрифту. Це допомагає зберігати пропорції інтерфейсу на різних екранах.

### 4. Віртуалізація списків

- Для відображення великих списків (наприклад, сотень працівників) використовується техніка **віртуалізації** (за допомогою `@tanstack/react-virtual`). Це означає, що в будь-який момент часу в DOM рендеряться лише ті елементи, які видимі на екрані (+ невеликий буфер). Це кардинально підвищує продуктивність, роблячи прокрутку та фільтрацію миттєвою навіть при роботі з тисячами записів.

Поєднання цих методів дозволяє створювати повністю адаптивний та високопродуктивний інтерфейс, який не просто "стискається", а розумно перебудовується, щоб забезпечити найкращий досвід користувача на будь-якому пристрої.
